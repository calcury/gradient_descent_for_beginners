Method Review: Least Squares Formula

In high school statistics, we learned that we can directly calculate the coefficients of the best-fit line y = kx + b using the formula:

$$k = \frac{\sum_{i=1}^n(x_i - \bar{x})(y_i - \bar{y})}{\sum_{i=1}^n(x_i - \bar{x})^2}, \quad b = \bar{y} - k\bar{x}$$

This method is very effective and fast for single-variable (1 feature) problems, with a computational complexity of only $O(n)$ (where $n$ is the number of samples). Let's test its performance on this dataset.

Identifying the Problem: True Limitations of the Least Squares Method

Although single-variable least squares fitting seems good, in reality, vehicle emissions depend not only on fuel consumption but also on multiple features such as engine displacement, number of cylinders, fuel type, and vehicle weight. As the number of features increases, the core problems of the least squares method become prominent:

Core Bottlenecks:

  • Explosive formula complexity: Only 2 formulas are needed for a single variable, a 3-variable system of equations for 2 features, and an 11-variable system of equations for 10 features—manual derivation and calculation become completely infeasible;
  • Exponential increase in computational complexity: The core of multi-dimensional least squares is solving the closed-form solution $$w = (X^TX)^{-1}X^Ty$$ where X is an n×d feature matrix (d is the number of features):
    • For d=10, matrix inversion complexity is O(d^3)=1000 operations, still acceptable;
    • For d=100, inversion complexity reaches 10^6 operations, and computation time begins to increase significantly;
    • For d=1000, inversion complexity skyrockets to 10^9 operations, requiring seconds or even minutes on ordinary computers, with a high risk of memory overflow;
  • Poor numerical stability: In high dimensions, X^TX is highly prone to "multicollinearity" (high correlation between features), leading to near-singular matrices that cannot be inverted. Even with pseudo-inverse solutions, results deviate significantly from the true optimal solution;
  • Inability to learn online: When new samples are added, least squares requires recalculating the entire matrix inverse, while iterative methods (such as gradient descent) only need minor parameter updates.

This is the core motivation for gradient descent: replace "one-time exact solution" with "iterative approximation" to handle high-dimensional problems with linear complexity O(nd), balancing efficiency and numerical stability.
